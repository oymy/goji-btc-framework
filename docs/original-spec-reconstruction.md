# Original Spec Reconstruction

## 目标

这份文档不是优化版，也不是弱化版。
它的目标只有一个：

**尽量忠实还原原版 V1 / V2 的定义。**

## 原版 V1

### 数据前提
- Funding Rate 必须使用 **全网 OI 加权值**
- 不接受单交易所作为等价替代
- 合约 CVD / 现货 CVD / OI / 订单簿 / 清算流动性来自 CoinGlass 体系

### 原版定位
- 逆向极值型
- 抓极端拥挤后的均值回归 / 挤压

### 原版触发核心
至少包括：
1. `FR <= -0.0040` 做多，或 `FR >= +0.0040` 做空
2. 五指标中 `3/5` 以上一致
3. 存在价格与 CVD 背离
4. 做多时要求现货 CVD “方向不恶化”

### 从案例补出的更细定义
V1 做多四条件，在 2026-04-02 案例中被写得最清楚：
1. FR <= -0.0040（全网 OI 加权）
2. 合约 CVD 持续下降
3. 价格跌幅小于合约 CVD 跌幅，形成 Higher Low / 背离
4. 现货 CVD 绝对值仍可为负，但方向不继续恶化，最好走平或修复

### 原版执行方式
- V1 触发后 **不追市价**
- 等待清算密集区入场
- 止损放在结构位，不用紧止损
- 用固定 TP，不用移动止损
- 触发后 10 分钟内决定，否则错过就等下一次

## 原版 V2

### 数据前提
- Funding Rate 使用原框架定义口径
- 现货 CVD 是核心指标
- OI / 订单簿 / 清算流动性共同参与评分

### 原版定位
- 趋势确认型
- 抓中性 funding 环境下的趋势共振

### 原版前提
1. `abs(FR) <= 0.0010`
2. 不存在明显现货/合约背离
3. 背离一旦出现，切回 V1 观察模式

### 原版评分
- Spot CVD: `±3`
- OI: `±1`
- 订单簿深度差值: `±1`
- 清算流动性: `±2`

### 原版结论阈值
- `>= +5` 做多
- `<= -5` 做空
- `-4 ~ +4` 观望

### 原版补充规则
- V1 与 V2 并行运行，但 **V1 优先**
- 评分 3-4 可灰度试探仓
- 评分加深到 5 再补仓

## 原版里最关键、也最难复现的部分

### 1. Funding 口径
这是最硬的要求：
- **全网 OI 加权 funding**
- 不是 Binance 单所
- 不是 OKX 单所
- 不是随便加权就等价

### 2. Spot / Futures CVD
原版对 CVD 的依赖很重：
- Spot CVD 决定“真资金是否确认”
- Futures CVD 决定“杠杆端是否在加压/平仓”

没有这两个字段，不能说高保真复现 V1/V2。

### 3. 清算密集区 / Heatmap
V1 执行层明确依赖：
- 等清算密集区
- 不追市价

这不是装饰规则，是原版执行的一部分。

### 4. 价格与 CVD 背离的判定
原文给出方向，但没有完全量化。
所以这里需要案例反推，不能只靠我自己编。

## 当前可以确定的事

### 可以确定
- 原版框架高度依赖 CoinGlass 全网聚合视角
- V1 不是“只看 funding 极值”
- V2 不是“只看 funding 中性”
- CVD 在原版里是核心，不是陪衬

### 还不能确定
- 背离的严格计算口径
- 3/5 的每个子项是否有固定权重边界
- 清算流动性评分的具体量化方法
- 案例里的实时判断是否还有未写进文档的口头规则

## 当前结论

如果目标是“证明复现”，那么标准必须是：

**是否和原版定义、原版数据口径、原版案例判断一致。**

而不是：

**是否在别的数据上也能跑出类似收益。**
