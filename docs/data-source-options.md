# Data Source Options

## 结论先说

如果目标是先做 1 年回测准备，建议分三档：

- A 档：结构化付费数据，质量最好
- B 档：多交易所公开数据，成本低，先能跑
- C 档：截图/人工读数，只适合学习，不适合主回测

## A 档：优先方案

### CoinGlass API / 同类聚合数据
优点：
- 更接近原框架口径
- 有聚合 funding / OI / liquidation 视角
- 对 V1 更友好

缺点：
- 付费
- 历史深度、字段可得性要单独确认
- CVD 未必能完整结构化拿到

适合：
- 想尽量保留原框架味道
- 接受付费

## B 档：务实方案

### Binance + Bybit + OKX 公开接口加权
优点：
- 免费或低成本
- funding / OI / OHLCV 好拿
- 足够做 first-pass 事件研究

缺点：
- 不完全等于全网聚合
- spot/futures CVD 聚合难度仍然高
- liquidation 需要弱化处理

适合：
- 先验证方向偏移
- 先跑弱化版 V1 / V2

## C 档：学习方案

### CoinGlass 页面截图 + 人工/视觉识别
优点：
- 贴近当前仓库日常使用方式
- 便于理解框架

缺点：
- 不稳定
- 难批量化
- 不适合作为 1 年主回测数据

适合：
- 学习
- 手工案例复盘

## 我建议的路线

### 第 1 步
先用 **B 档** 做弱化版研究：
- Price
- Funding
- OI
- 简化版 spot/futures proxy

### 第 2 步
如果 first-pass 有点意思，再补 **A 档**：
- 付费聚合数据
- 更贴近原框架

### 第 3 步
C 档保留给学习和人工验证，不做主回测依据。

## 当前推荐

如果你现在还在学习阶段，最稳的是：

**先用 B 档把研究链路跑通，再决定是否为 A 档付费。**
