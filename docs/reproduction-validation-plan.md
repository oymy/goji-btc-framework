# Reproduction Validation Plan

## 目标

证明的不是“优化版有效”，而是：

**原版 V1 / V2 已被正确复现。**

## 验证标准

### Level 1: 文档一致
复现版规则是否和原文档一致。

### Level 2: 案例一致
文档里列出的案例，复现版是否能在相同时间点得出相同方向判断。

### Level 3: 数据一致
复现版使用的数据口径，是否接近原作者当时实际使用的数据口径。

### Level 4: 使用者验收
原作者或原框架使用者是否认可“这就是原版意思”。

## 当前最关键的验证对象

### V1 案例
- 2026-03-31 轧空案例
- 2026-04-02 关税冲击日
- 2026-04-02 V1 做多完整信号确认

### V2 案例
- 2026-04-01 FR 灰色地带看跌背离

## 当前工作顺序

1. 先写原版规格说明书
2. 再做案例字段清单
3. 再做逐案例一致性核对表
4. 最后才谈“是否复现成功”

## 当前结论

如果没有：
- CoinGlass 同口径数据
- 案例时点数据快照
- 原作者/使用者验收

那最多只能说：

**接近原版，不能说已证明复现。**
