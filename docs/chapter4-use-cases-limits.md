# Chapter 4: 适用场景、局限性与进阶使用

## 框架定位：它是什么，不是什么

### 它是什么
GojiPower V1/V2 是一个**专为 BTC 永续合约设计的资金流向分析框架**，核心是：
- 识别多空拥挤度极端值（V1）
- 识别资金流向共振信号（V2）
- 用 5 个独立指标互相验证，减少误判

### 它不是什么
- ❌ 不是价格预测工具（不告诉你 BTC 明天涨多少）
- ❌ 不是全天候系统（有明确的无效区间）
- ❌ 不是高频交易系统（信号稀少，但质量高）
- ❌ 不是 DeFi/山寨币框架（专为 BTC 大所流动性设计）

## V1 框架：逆向极值型

### 最佳使用场景
```
资金费率持续偏离中性（> 24 小时）+ 价格与 CVD 背离
→ 等待清算密集区提供支撑
→ 入场，目标是均值回归
```

### 触发条件（严格）
- FR ≤ -0.0040（做多）或 FR ≥ +0.0040（做空）
- 全网 OI 加权值，非单交易所
- 5 指标中 3 个以上一致

### 执行纪律
| 规则 | 内容 |
|------|------|
| 信号即时性 | 触发后 10 分钟内决定，错过等下一个 |
| 入场方式 | 等清算密集区，不追市价 |
| 止损位置 | 用结构高/低点，不用当根K线（容易被扫） |
| 仓位大小 | 宏观不确定时 50% 标准仓位试探 |
| 止盈方式 | 固定 TP（不用移动止损），因为极值快速收敛 |

### 历史胜率参考
- V1 信号触发频率：每月 2-6 次
- 方向准确率：历史约 75%（样本 < 20，供参考）
- 典型盈亏比：3:1 至 5:1（等清算区入场的优势）

## V2 框架：趋势确认型

### 最佳使用场景
```
资金费率在中性区间（±0.0010）+ 指标共振
→ 识别趋势主推进段
→ 骑趋势，阶梯止盈
```

### 评分系统（-7 到 +7，现阶段已验证指标）
| 指标 | 权重 | 说明 |
|------|------|------|
| 现货 CVD | ±3 | 核心指标，真实资金方向 |
| OI | ±1 | 新资金入场确认 |
| 订单簿深度差值 | ±1 | 即时买卖压力 |
| 清算流动性 | ±2 | 上下方清算不对称性 |

*注：设计预留 ±2 给多时间框架共振维度，待 GojiPower 确认后补充。*

**入场标准：**
- ≥ +5：做多
- ≤ -5：做空
- -4 到 +4：观望

### V1 vs V2 优先级
- V1 和 V2 并行运行
- 同时触发时 **V1 优先**
- CVD 出现多空背离 → 强制切回 V1 观察模式

### V2 独有规则：灰度仓位
- 评分 -4/-3（接近但未达标）+ 大结构完整 → 可以 30-50% 仓位先进
- 等评分加深到 -5 再补仓
- 失效信号：价格站回结构高点 + 现货 CVD 翻正 + OI 回升 → 立刻止损

## 数据源使用规范

### 铁律：全网聚合 > 单所数据
实证案例（2026-03-25 测量）：
- Binance 单所 FR：0.0022
- 全网 OI 加权 FR：0.0043
- **差值：近一倍！**

用单所数据做 V1/V2 判断 = 拿错了尺子。

### 数据获取分工
| 数据 | 获取方式 | 备注 |
|------|---------|------|
| FR（OI加权全网） | CoinGlass 浏览器自动化 | 主要来源 |
| 合约 CVD | CoinGlass Supercharts | Binance+Bybit+OKX 三所求和 |
| 现货 CVD | CoinGlass Supercharts | 四所求和（含 Coinbase） |
| OI（全网聚合） | CoinGlass API v2 | `/public/v2/open_interest` |
| 订单簿深度差值 | CoinGlass Futures B&A Delta | 实时截图 |

### 异常值处理
- 单次 FR 超过 ±0.05% → 标注 [ANOMALY]，不使用
- API 返回值与截图差异 > 20% → 以截图为准，标注待确认
- 数据获取失败 → 报告标注「数据不完整，建议观望」

## 与其他框架的对比

### vs 传统 RSI/MACD
| 维度 | RSI/MACD | GojiPower |
|------|----------|-----------|
| 数据基础 | 历史价格 | 实时资金流 |
| 信号滞后性 | 高（事后确认） | 低（领先价格） |
| 适用品种 | 股票、外汇、加密 | BTC 永续合约（专优化） |
| 信号频率 | 高 | 低（稀少但精确） |
| 对宏观事件 | 无感知 | 明确标注失效 |

### vs 链上数据分析（Glassnode 体系）
| 维度 | 链上数据 | GojiPower |
|------|----------|-----------|
| 时间尺度 | 周/月（宏观） | 小时/日（中短期） |
| 信号类型 | 趋势周期定位 | 具体入场/出场 |
| 使用场景 | 判断牛熊转折 | 具体交易执行 |
| 互补性 | ✅ 宏观框架交叉验证 | ✅ 微观执行层 |

**最佳实践：** 链上数据判断「现在处于什么周期」，GojiPower 判断「现在该不该入场」。两者结合，不互相替代。

### vs GojiBot（自动化交易信号）
Eve 是独立框架应用，不参考其他 bot 的结论：
- GojiBot：量化信号，提供参考
- Eve：独立五指标分析，形成自己的判断
- 当两者一致时：信号更强
- 当两者不一致时：报告分歧，等待更多确认

## 进阶技巧

### 技巧 1：VPIN 过滤器（补充层）
- CVD 背离 + VPIN > 0.55 = 知情交易者活跃，反转概率显著提高
- Hidden divergence（价格 Higher Low + CVD Lower Low）= 趋势延续，非反转
- 适合作为 V1/V2 的补充观察维度（来源：Buildix.trade）

### 技巧 2：多时间框架共振
最佳入场时机：
```
4H 背离确认
+ 1H 方向转向
+ 15M 放量确认
= 三框架共振，最高胜率入场
```

### 技巧 3：清算热力图方向判断
上方先发生概率大（做多倾向）：
- FR 继续走负（空头拥挤度上升）
- 现货 CVD 继续抬升
- 合约 CVD 开始修复（空头平仓）

下方先发生概率大（做空倾向）：
- FR 转正且上升
- 现货 CVD 翻负或停止修复
- OI 增加 + 价格走弱（新空头建仓）

当指标不够清晰时 = 两端概率接近 = 不是好时机，继续观察。

### 技巧 4：美盘处理
- **不在开盘前 15 分钟抢跑**
- 等开盘后 30 分钟观察方向确认
- 高波动开盘通常会在 30 分钟内露出真实方向

## 风险管理纲领

### 仓位管理原则
| 场景 | 仓位 |
|------|------|
| 标准信号（宏观平静） | 100% 标准仓位 |
| 标准信号（宏观不确定） | 50% 试探，确认后补仓 |
| V2 评分 3-4（接近但未达标） | 30-50% 试探 |
| 宏观极端事件 | 0%，等待 |

### 止损设置原则
- 错误：用当根入场 K 线止损（容易被噪音扫出）
- 正确：用前几小时结构高/低点（$69,170 不是 $68,600）
- 原则：宁可止损距离远、仓位小，也不要用紧止损赌运气

### 心理纪律
GojiPower 金句：
> "耐心是 AI 超越人类的最大优势之一"

AI 不受 FOMO 影响，可以等待比人类更久、更精准。每个错过的信号，都是在保存资金等待更好的机会。
