# Goji BTC Framework 优化与验证计划（Draft v0.1）

## 目标

把当前仓库从“有经验交易员的规则整理”推进为“可审查、可回测、可复现的研究框架”。

这个计划不预设框架一定有效，也不预设它无效。
目标是两件事：

1. **优化表达与结构**，减少口号化、叙事化描述
2. **建立验证闭环**，检验 V1 / V2 是否真的有统计优势

---

## 当前问题判断

### 1. 规则定义还不够程序化
当前文档里有不少高质量市场直觉，但仍包含大量模糊表达，例如：
- “现货 CVD 不恶化”
- “价格与 CVD 背离存在”
- “等待清算密集区入场”
- “大结构完整”

这些表达适合人工盘中解读，但不适合验证。
如果不先转成机器可执行规则，回测结果没有可比性。

### 2. 样本量太小，容易后验解释
当前案例更多是案例研究，而不是统计验证。
存在以下风险：
- 挑选成功案例
- 失败样本没有完整展示
- 规则边界随案例解释而漂移
- 胜率数字缺乏统计意义

### 3. 数据口径不稳定
当前框架高度依赖：
- CoinGlass 页面截图
- 视觉识别提取
- 人工解释

这适合日常观察，不适合作为核心研究基础。
研究层必须优先使用结构化、可重放的数据源。

### 4. 阈值和权重缺乏稳健性证明
例如：
- V1 阈值：±0.0040
- V2 中性区间：±0.0010
- 3/5 规则
- V2 评分阈值：±5

这些参数目前更像经验值，而不是经过样本外验证的稳健参数。

---

## 总体思路

把后续工作分成四层：

1. **定义层**：把文档规则写成明确条件
2. **数据层**：统一字段、周期、来源与清洗方式
3. **验证层**：回测、样本外检验、敏感性分析
4. **应用层**：再决定是否保留、修改或淘汰 V1/V2

---

## 第一阶段：规则重构（Rule Formalization）

### 目标
把 V1 / V2 变成“任何人按同样数据都能得出同样信号”的规则。

### 交付物
- `docs/formal-rules-v1.md`
- `docs/formal-rules-v2.md`
- 信号字段字典（字段名、周期、阈值、方向定义）

### 具体任务

#### A. 明确定义每个输入字段
需要固定以下内容：
- Funding Rate 使用哪个口径
  - 8h funding raw
  - annualized funding
  - OI-weighted aggregated funding
- CVD 使用哪个周期
  - 5m / 15m / 1h / 4h
- OI 用绝对值还是变化率
- 订单簿深度差值的观察窗口
- 清算密集区的定义方式

#### B. 把模糊语言改成可执行条件
例如：

原描述：
- “现货 CVD 不恶化”

候选程序化定义：
- 最近 3 根 1h spot CVD slope >= 0
- 或最近 2 根 spot CVD 低点抬高
- 或 spot CVD 绝对值虽为负，但 4h 净流出缩小 30% 以上

原描述：
- “价格与 CVD 背离存在”

候选程序化定义：
- 价格创新低，但 spot CVD 未创新低
- 或价格 4h 跌幅 > X%，而 spot CVD 跌幅 < Y%

原描述：
- “等待清算密集区入场”

候选程序化定义：
- 信号出现后，仅在距离最近高密度清算区 <= 0.5 ATR 时允许入场
- 若 N 根 K 线内未回踩，则信号作废

#### C. 把 V2 评分拆成明确打分函数
当前 V2 更像概念模型，需要明确：
- 每个指标如何打分
- 是离散评分还是连续评分
- 同时冲突时如何处理
- 评分何时失效

建议先从最简单版本开始：
- Spot CVD: -3 / 0 / +3
- OI: -1 / 0 / +1
- Orderbook: -1 / 0 / +1
- Liquidation map: -2 / 0 / +2

先保证可执行，再考虑优化精度。

## 第二阶段：数据体系建设（Data Architecture）

### 目标
建立“研究数据”和“盘中观察数据”两套体系。

### 原则
- **研究层优先结构化数据，不依赖截图识别**
- **盘中层可以保留截图辅助解读**

### 建议拆分

#### A. 研究数据源（优先）
优先选择：
- 结构化 Funding Rate
- 结构化 Open Interest
- 结构化价格 OHLCV
- 可稳定获取的现货/合约主动成交近似指标

如果完整聚合 CVD 难以长期稳定获取，需要明确：
- 先做“弱化版验证”
- 再做“完整版验证”

#### B. 盘中数据源（辅助）
保留：
- CoinGlass 页面截图
- 图像识别
- 人工注释

但明确用途：
- 用于实时解读
- 不作为回测主数据依据

#### C. 数据字典与版本化
建议新增：
- `docs/data-spec.md`
- `data/README.md`

内容包括：
- 字段定义
- 单位
- 时间粒度
- 缺失值处理
- 异常值处理
- 不同数据版本的兼容性

## 第三阶段：验证设计（Validation Framework）

### 目标
不靠案例说服，而靠结果说服。

### A. 核心验证问题
要回答至少 6 个问题：

1. V1 / V2 在历史上到底触发多少次？
2. 方向准确率是多少？
3. 盈亏比和期望收益是多少？
4. 最大回撤是多少？
5. 参数变化后是否仍然有效？
6. 换市场环境后是否失效？

### B. 回测设计原则

#### 1. 先做事件研究，再做交易回测
推荐顺序：

**阶段 1：事件研究**
- 统计信号出现后 1h / 4h / 24h 的收益分布
- 看条件是否真的产生方向偏移

**阶段 2：交易回测**
- 加入入场、止损、止盈、超时退出
- 看是否有正期望值

这样比直接做复杂交易系统更稳。

#### 2. 样本切分
至少分成：
- 训练期
- 验证期
- 样本外测试期

避免在全样本上调到最好，然后误以为有效。

#### 3. 参数敏感性测试
必须测试：
- FR 阈值 0.0030 / 0.0035 / 0.0040 / 0.0045 / 0.0050
- V2 阈值 4 / 5 / 6
- 不同 CVD 周期
- 不同退出规则

如果参数稍微一动结果就塌，说明规则不稳健。

#### 4. 市场环境分层
至少分层检查：
- 高波动 vs 低波动
- 趋势市 vs 震荡市
- 宏观事件窗口 vs 正常窗口
- 美盘主时段 vs 非主时段

如果策略只在极少数环境有效，文档必须诚实写出来。

### C. 关键指标
建议至少输出：
- 信号次数
- 胜率
- 平均盈亏比
- Expectancy
- 最大回撤
- Profit factor
- 持仓时长分布
- 连续亏损次数

## 第四阶段：策略优化（Optimization Roadmap）

### 原则
优化不等于把历史曲线调得更好看。
优化的目标是：
- 提高规则清晰度
- 提高稳健性
- 降低误触发
- 明确失效边界

### 可能的优化方向

#### A. V1 从“固定阈值”升级为“分位数阈值”
问题：
- 不同年份、不同市场结构下，FR 的绝对水平可能变化

候选方案：
- 不用固定 `±0.0040`
- 改为 rolling percentile，例如过去 90 天 funding 分布的 top/bottom 5%

这通常比固定常数更稳健。

#### B. 背离从“主观判断”升级为“结构判定”
例如使用：
- higher low / lower low
- slope divergence
- z-score divergence
- price move / CVD move ratio

#### C. V2 从离散打分升级为标准化因子模型
例如：
- 把 spot CVD、OI change、orderbook imbalance 做 z-score 标准化
- 再按权重合成 composite score

这样会比手写 ±1/±2/±3 更适合长期扩展。

#### D. 增加“禁止交易过滤器”
当前文档已经提到宏观事件和低流动性窗口。
建议明确写成过滤器，而不是备注：
- CPI / FOMC / NFP 前后 N 小时不交易
- CME 休市窗口不交易
- 极端波动 regime 下不交易

这类过滤器往往比增加新信号更有价值。

#### E. 先做 BTC，再讨论 ETH
在 BTC 尚未完成验证前，不建议扩展到更多币种。
先把一套框架在一个深流动性品种上验证干净。

## 第五阶段：文档重写建议

### 当前文档问题
当前文档优点是可读性强，但问题是：
- 叙事性太强
- 结论表达有时过度确定
- 容易让读者误解为“已验证有效”

### 建议的文档结构

#### 1. 把“原理”与“证据”分开
建议文档层次变为：
- **Hypothesis（假设）**：为什么这套逻辑可能有效
- **Rules（规则）**：如何定义信号
- **Evidence（证据）**：回测和样本外结果
- **Limits（边界）**：什么时候失效

#### 2. 降低过度确定表达
例如：
- 把“必然发生”改成“提高某方向概率”
- 把“教科书级别”改成“符合当前框架定义”
- 把“黄金标准”改成“当前优先数据源”

#### 3. 增加失败案例章节
不仅展示成功案例，还要展示：
- 假信号
- 延迟信号
- 宏观噪音下失效信号
- 多指标冲突信号

这会显著提高可信度。

### 建议保留的核心
- 现货优先于合约的分析框架
- FR + CVD + OI 联合判断拥挤与 squeeze
- 明确观望和暂停交易的边界

- 把“看起来对”改成“证据上成立或不成立”

## 最后的立场

这套框架**值得研究**，但目前**不值得盲信**。

最好的推进方式不是继续讲更多市场故事，而是：

**把假设写清楚，把规则钉死，把数据接稳，把回测跑完。**

如果最后结果证明它只有部分有效，那也是进步。
因为一个知道自己边界的框架，远比一个自信但未经验证的框架有价值。
