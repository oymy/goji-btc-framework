# Funding Threshold Research

## 数据来源

本轮先接入 `supervik/historical-funding-rates-fetcher` 的 BTC-USDT funding CSV。

覆盖交易所：
- Binance
- Bybit
- dYdX
- Gate
- HTX
- Kucoin
- MEXC

用途：
- 研究 funding 分布
- 评估固定阈值是否合理
- 为 percentile 阈值提供依据

## 关键发现

### 1. 固定阈值 `±0.0040` 不适合作为通用阈值
多所历史结果显示：
- Binance max: `0.003`
- Bybit max: `0.00375`
- Gate max: `0.00375`
- Kucoin max: `0.00375`
- MEXC max: `0.0013`
- dYdX max: `0.00059`

这意味着：

**`+0.0040` 在很多数据源里几乎不会出现，甚至超过历史样本极值。**

### 2. 更现实的极端区间通常落在 p95 ~ p99
例如：

#### Binance
- p95: `0.00061936`
- p99: `0.00123546`

#### Bybit
- p95: `0.00081846`
- p99: `0.00201317`

#### Gate
- p95: `0.000671`
- p99: `0.00152`

#### Kucoin
- p95: `0.000706`
- p99: `0.001443`

### 3. 不同交易所差异明显
- dYdX 整体 funding 波动显著更小
- Binance / Bybit / Gate / Kucoin 更接近“高波动尾部”特征
- 所以单一固定阈值天然不稳

## 当前结论

### 对原框架的影响
原框架把 `±0.0040` 写成固定触发条件，问题很大：
- 它强依赖特定数据口径
- 它不具备良好的跨所迁移性
- 它不适合作为 B 档研究的统一阈值

### 当前更合理的做法
在 B 档或单所研究里：
- 优先用 percentile threshold
- 或按交易所单独标定阈值

## 当前建议阈值方向

### 学习 / first-pass
- 用 p90 / p10 做候选极端
- 用 p95 / p05 做强极端

### 后续回测
- 比较 p90 / p95 / p99 三档
- 看哪个区间在 24h 维度最有解释力

## 暂时结论

**funding 极值这件事本身可能有用。**
但“极值”应该是：

**相对该数据源自身分布的极值，**
而不是写死成一个跨环境不变的常数。
