# First-Pass Findings

## 当前环境

- Binance API: 451
- Bybit API: 403
- OKX API: 可用（通过 Clash 代理）

所以当前研究降级为：

**OKX 单所弱化版 first-pass。**

## 原始阈值测试结果

按原文档 V1 阈值：
- `funding_rate <= -0.0040`
- `funding_rate >= +0.0040`

在当前 OKX 单所 funding 数据里：
- `min = -0.000071`
- `max = 0.000100`

结果：
- **0 次触发**

### 结论

原文档里的 funding 阈值不能直接迁移到单所口径。
它明显依赖特定聚合数据源。

## 分位数版 V1 first-pass

为了继续验证，改成单所分位数阈值：
- 低阈值：最近样本 bottom 10%
- 高阈值：最近样本 top 10%

本次样本阈值：
- `low_threshold = -0.0000428587`
- `high_threshold = 0.0000757378`

### 结果
- 总信号数：`19`
- LONG：`10`
- SHORT：`9`

### 平均后续收益

#### LONG
- 4h 平均：`-0.13%`
- 24h 平均：`+1.26%`

#### SHORT
- 4h 原始价格收益：`-0.14%`
- 24h 原始价格收益：`-0.75%`
- 若按做空方向理解：
  - 4h：`+0.14%`
  - 24h：`+0.75%`

## 当前解读

1. 原始固定 funding 阈值不可直接跨源复用
2. 改成单所分位数阈值后，V1 有初步触发能力
3. 当前样本下，V1 更像 **24h 级别均值回归**，不是强 4h 反转信号
4. 这还只是 funding 单因子 first-pass，不是完整 V1

## 目前还缺什么

当前结果还不能当成正式结论，因为还缺：
- OI 历史序列
- 更稳定的 flow proxy
- 更长 funding 历史
- 手续费 / funding cost / 出场规则

## 暂时结论

如果只看当前 OKX 单所弱化版结果：

**V1 不是没东西，但原文档写法明显过度绑定特定数据口径。**

更合理的方向是：
- 单所环境下用 percentile 阈值
- 先继续做弱化版事件研究
- 再决定要不要升级到更贵的数据源
