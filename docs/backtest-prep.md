# Backtest Prep

## 目标

不是现在就宣称能跑出靠谱回测。
这份文档的目标是定义“开始做 1 年回测前，必须先补齐什么”。

## 一年回测，能做
但要分两版：

### 版本 A：弱化版
用：
- Price
- Funding
- OI
- 简化 CVD

目的：
- 先验证 V1 / V2 是否大致有方向偏移
- 不急着做完整交易系统

### 版本 B：完整版
再加入：
- 更稳定的聚合 CVD
- 更明确的 liquidation context
- 更明确的 entry / exit rules

目的：
- 看是否存在正期望
- 看回撤和稳健性

## 建议顺序

### Step 1
先做事件研究：
- 信号后 1h 收益
- 信号后 4h 收益
- 信号后 24h 收益

先回答：信号后是否真的偏某个方向。

### Step 2
再做简化交易回测：
- 固定止损
- 固定止盈
- 超时退出

### Step 3
最后才做参数敏感性：
- FR 阈值
- V2 打分阈值
- 不同周期

## 回测前必须固定的东西

### 1. 时间周期
建议首版固定：
- 主周期 1h
- 背景 4h

### 2. V1 候选规则
至少要固定：
- FR 极值阈值
- Spot CVD “不恶化”的定义
- OI 下降的定义
- 3/4 或 4/4 的触发方式

### 3. V2 候选规则
至少要固定：
- FR 中性区间
- Spot CVD 评分规则
- OI / orderbook / liquidation 的简化评分
- >=5 / <=-5 是否保留

### 4. 收益统计方式
至少固定：
- 开仓价格取 close 还是 next open
- 手续费是否计入
- funding cost 是否计入
- 同时信号如何处理

### 5. 失效和剔除规则
至少固定：
- 宏观事件窗口是否跳过
- 数据缺失是否跳过
- 低质量数据是否降级或剔除

## 当前最务实的路线

先别追求“完整一年策略回测”。
先做：
1. 文档收紧
2. 数据口径固定
3. 事件研究
4. 再上交易回测

## 结论

1 年回测可以做。
但前提不是继续讲故事，前提是先把规则和数据钉住。

当前仓库已经开始往这个方向补基础设施。
