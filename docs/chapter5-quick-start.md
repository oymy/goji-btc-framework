# Chapter 5: 新手入门指南

## 先问三个问题

**Q1: 我需要真的去交易吗？**
不需要。这个框架可以只用来「看懂市场在发生什么」，不一定要下单。

**Q2: 我需要懂技术分析吗？**
不需要。这套框架不看 K 线，不看均线，不看 RSI。它看的是「钱在流向哪里」。

**Q3: 这套框架安全吗？**
任何交易都有风险。这套框架的价值是：减少误判，提高决策质量。它不保证盈利，但提供了比随机猜测更好的判断依据。

## 三分钟理解核心概念

### 1. 什么是永续合约？

普通买 BTC = 你真的拥有了这枚比特币。

永续合约 = 你在赌 BTC 的价格涨跌，不需要真正买入。可以做多（赌涨），也可以做空（赌跌），还可以加杠杆（放大收益和损失）。

**关键特点：** 合约市场的交易量通常是现货市场的 5-10 倍。这意味着价格走向，很大程度上由合约市场决定。

### 2. 什么是资金费率？

合约市场有个自动平衡机制：每 8 小时，多头和空头之间互相支付费用。

- 做多的人太多 → 多头付费给空头（资金费率为正）
- 做空的人太多 → 空头付费给多头（资金费率为负）

**为什么重要？** 当费率极端（比如空头每 8 小时要付 0.4%），空头的持仓成本变得极高，他们被迫平仓，这会推动价格反向运动。

### 3. 什么是 CVD？

想象你在观察一个市集：
- 有人以高于标价的价格抢购苹果（主动买入）
- 有人以低于标价的价格甩卖苹果（主动卖出）

CVD 就是记录这两种行为的差值：主动买入越多，CVD 越高。

**为什么有两个 CVD？**
- 现货 CVD：真实买入/卖出 BTC（用真金白银）
- 合约 CVD：合约方向上的买卖（可以加杠杆）

**现货 CVD 更可信**，因为它代表真实资金的行动，而非杠杆赌注。

## 一个比喻：读懂市场的「温度计」

把 BTC 市场想象成一个浴缸：

```
热水 = 真实买盘（现货 CVD 上升）
冷水 = 真实卖盘（现货 CVD 下降）
水泡 = 杠杆合约（可以人为制造噪音）
水温 = 资金费率（反映当前多空拥挤程度）
```

GojiPower 框架不看水面是什么颜色（价格涨跌），而是测量：
1. 热水还是冷水在流入？（现货 CVD 方向）
2. 水温是否极端？（资金费率是否达到 ±0.0040）
3. 水泡是在增加还是消失？（OI 在涨还是跌）

**当水温极端冷（大量做空）+ 热水在悄悄流入（现货积累）= 温度即将反弹。**

## 两种信号模式

### V1 模式：「极端时逆向」

适用场景：市场情绪极端，大多数人都在同一方向押注。

**比喻：** 当所有人都在抛售时，聪明钱在低价买入。

触发信号：
- 资金费率达到极端（超过 ±0.0040）
- 现货买盘在悄悄积累
- 5 个指标中 3 个以上一致

**这时候做什么：** 逆向而行（大家都做空时考虑做多）

### V2 模式：「确认后顺势」

适用场景：资金费率中性，但多个指标显示方向一致。

**比喻：** 不是在逆流而行，而是判断潮水开始流向哪里，然后跟上去。

触发信号：
- 资金费率在 ±0.0010 之间
- 5 个指标综合评分 ≥ ±5 分

**这时候做什么：** 顺趋势方向操作

## 什么时候什么都不做？

这套框架最重要的规则之一：**不确定时，观望是最好的选择。**

以下情况 Eve 会明确宣布「观望」：
- 指标互相矛盾
- 重大新闻事件发生（Trump 讲话、战争、CPI 数据）
- 节假日（Good Friday、圣诞节）
- 数据无法获取

**GojiPower 说：** 「最好的交易有时候是不交易。」

## 如何使用 Eve 的分析报告

Eve 在 #trading 频道发布的报告格式：

```
BTC 监测 - 2026-04-09 09:00 PT

当前框架：V1 观察中

五指标
1. 资金费率：-0.0038（接近阈值，未触发）
2. 合约 CVD：-$45M（↓ 持续下降）
3. 现货 CVD：-$12M（→ 走平，不再恶化）
4. OI：$48.2B（↓ 去杠杆中）
5. 订单簿：+1200 BTC（买墙略强）

信号评分：3/5（达标）
框架状态：V1 边缘，等待 FR 突破 -0.0040

结论：观望，密切关注 FR 变化
失效条件：FR 反弹至 -0.0015 以上 = 信号取消
```

**怎么读这个报告：**
- 「信号评分 X/5」→ 3 以上开始关注，5 是最强信号
- 「框架状态」→ 当前处于 V1/V2 哪个模式
- 「结论」→ 直接告诉你该做什么（做多/做空/观望）
- 「失效条件」→ 什么情况下这个判断失效

## 开始使用（三步）

### Step 1：安装 Skill
```bash
# OpenClaw 用户
/plugin install goji-btc-framework

# Claude Code 用户
npx skills add goji-btc-framework
```

### Step 2：连接数据源
Eve 会自动从 CoinGlass 获取数据，无需配置 API key（使用浏览器自动化）。

可选配置 CoinGlass API key 以获得更稳定的数据：
```json
{
  "COINGLASS_API_KEY": "你的 API key"
}
```

### Step 3：触发分析
```
/trading-scan           → 运行完整五指标扫描
/trading-scan --quick   → 快速扫描（仅看 FR 和 CVD）
/trading-v1             → 检查 V1 信号状态
/trading-v2             → 检查 V2 信号状态
/trading-explain        → 用简单语言解释当前市场状态
```

## 常见问题

**Q: 这套框架适合做短线还是长线？**
A: 中短线（数小时到数天）。基于 1H/4H 数据，不适合秒级交易，也不是月级投资框架。

**Q: 只看 BTC 吗？**
A: 主要针对 BTC 永续合约优化。ETH 部分适用，山寨币流动性较低，效果未经验证。

**Q: 我需要多少资金？**
A: 框架本身不设资金门槛。但建议：单笔交易亏损不超过账户 2%，每日最大亏损不超过 5%。

**Q: Eve 会帮我自动下单吗？**
A: 当前版本是分析工具，提供信号和建议，执行需要人工确认。自动执行功能在规划中，需要 Hyperliquid 或 OKX API 集成。

**Q: GojiPower 是谁？**
A: GojiPower 是 #trading 频道中持续提供高质量交易分析的专家，这套 V1/V2 框架是在他的指导和验证下建立的。
